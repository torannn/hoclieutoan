Test bảng xét dấu / biến thiên

So sánh 3 path: (a) manual TkzTab-like, (b) auto sign-table với mảng expressions, (c) auto variation-table với chỉ f (classifier JS tự tính f' + nghiệm). Tất cả đều đi qua drawVariationTableFromData → style đồng nhất.

A. Manual — bảng xét dấu 1 dòng

A1. f(x) = x² − 4x + 3

A2. BBT cổ điển — f'(x), f(x) 2 dòng

Kỳ vọng: style đều chữ (Times, MathJax), padding cạnh scale theo cellW, arrow inset thoáng hơn, giá trị cực đại/cực tiểu không đè arrow.

B. Auto — bảng xét dấu (auto: true, expressions)

B1. Tam thức x² − 4x + 3

B2. Phân tử × mẫu số

Nhân tử mẫu (forbidden) được đánh dấu double-bar.

B3. Đa thức có nghiệm kép — x³ − 3x + 2 = (x−1)²(x+2)

Kỳ vọng: tìm đúng nghiệm kép tại x=1 và nghiệm đơn x=−2 (bisection cũ hay bỏ lỡ nghiệm kép).

B4. Đa thức bậc 5 — x⁵ − x = x(x²−1)(x²+1) = x(x−1)(x+1)(x²+1)

Durand–Kerner tìm 3 nghiệm thực {−1, 0, 1}, lọc 2 nghiệm phức.

C. Auto — bảng biến thiên (kind: variation, chỉ cho f)

C1. f(x) = x³ − 3x + 2

Classifier nhận dạng đa thức → f'(x) = 3x² − 3 → nghiệm ±1 → BBT đầy đủ với arrow + cực trị.

C2. f(x) = x⁴ − 2x²

C3. Hàm phân thức — f(x) = (x² + 1)/(x − 1)

Classifier split P/Q → f' = (2x(x−1) − (x²+1))/(x−1)² = (x²−2x−1)/(x−1)². Tử f' có nghiệm x = 1 ± √2; mẫu Q thêm x=1 vào forbiddenValues. Kỳ vọng: 4 cột (−∞, 1−√2, 1, 1+√2, +∞) với bar kép tại x=1.

C4. Phân thức bậc 2/bậc 1 khác — f(x) = (x² − 4)/(x + 1)

D. Khi cần SymPy — hàm transcendental & nghiệm symbolic

D1. f(x) = sin(x) − x/2 trên [−5, 5]

Classifier fail (không phải poly/rational) → auto fallback qua SymPy (Pyodide) nếu engine:"sympy"; bỏ trống engine thì dùng bisection numeric thuần JS.

D2. BBT với nghiệm vô tỉ — f(x) = (x²+1)/(x−1) (SymPy)

Kỳ vọng: nhãn x hiển thị 1-\sqrt{2}, 1+\sqrt{2}; cực đại/cực tiểu f(x) hiện 2-2\sqrt{2}, 2+2\sqrt{2} (thay vì xấp xỉ thập phân). So sánh với C3 đi bằng JS fast-path.

D3. BBT transcendental — f(x) = e^(−x²)

Giới hạn ở ±∞ = 0, cực đại tại x=0, f(0)=1. SymPy trả LaTeX đúng cho cả giá trị hữu hạn lẫn giới hạn.

E1. Tiered (mặc định) — C1

Độ cao giá trị phản ánh thứ tự độ lớn.

E2. Flat — cùng dữ liệu

Tất cả giá trị trên đường midline — hàng variation gọn hơn.
